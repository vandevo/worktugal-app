## 2. Key Decisions Made

Technical, stack, automation, schema
Decision: Freeze table name as compliance_diagnostics with raw_answers jsonb instead of individual columns. Nullable user_id. Versioning columns (diagnostic_version, ruleset_version).
Impact: Schema stable as questions evolve. No migration needed when adding new quiz fields. Anonymous submission possible without auth.
Files or IDs: supabase/migrations/20260305200000_create_compliance_diagnostics_table.sql

Technical, stack, automation, schema
Decision: Declarative trap rules with condition objects evaluated by engine, not embedded functions. Each rule includes source_url, legal_basis, penalty_range, last_verified.
Impact: Country expansion becomes config, not code. Source citations power both product credibility and Parallel.ai monitoring targets.
Files or IDs: src/lib/diagnostic/rules/portugal.ts (RULESET_VERSION: portugal_v1)

Strategic, ICP, monetization, positioning
Decision: Path A (retail wedge first, with Path B architecture). Build direct to consumer product but design engine as country modular and embeddable from day one.
Impact: Revenue validation within 21 days. Retail proof becomes the pitch for B2B infrastructure licensing to relocation firms.
Proof or metric: First paid 29 EUR scan within 21 days of Phase 2 launch.
