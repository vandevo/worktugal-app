## 2. Key Decisions Made

Schema: compliance_diagnostics table frozen. raw_answers jsonb (not individual columns). Nullable user_id (no auth required for quiz). diagnostic_version and ruleset_version columns for data continuity.

Auth model: Anonymous submission with email gate. No Supabase Auth required before quiz completion. user_id linked retroactively if user creates account later. ChatGPT recommended auth-first; this was overruled because it kills conversion.

Question strategy: Phase 1 uses the original 12 Setup Check questions exactly (preserves benchmark continuity with 865 records). Phase 2 will add 6 exposure detection questions.

Trap rules: Declarative config objects with condition matching (AND between fields, OR via arrays). Not embedded functions. Each rule has source_url, legal_basis, penalty_range, last_verified.

Monetization: 3-layer ladder. Layer 1: 29 EUR Compliance Risk Scan (Phases 1-4). Layer 2: 99-149 EUR Clarity Call (Phase 6). Layer 3: 1K-5K EUR/month B2B infrastructure licensing (Phase 7).
