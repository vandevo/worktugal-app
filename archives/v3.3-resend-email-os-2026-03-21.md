## 1. Thread Overview

Full replacement of Make.com transactional email layer with Resend across the Worktugal app. Make.com was used for 5+ webhook-triggered email flows — slow UI, unnecessary middleware, no visibility. Resend is now the single transactional layer. Listmonk handles campaigns. Telegram bot handles real-time internal alerts.

Stack drivers: worktugal-app on Supabase (project ref yxkwvindawbzzsvbxcrn). All email logic lives in Supabase edge functions (Deno). Frontend is React/Vite hosted on Cloudflare Pages.

This session also retired the tax-checkup flow (legacy), completed the diagnostic compliance email system (active), and wired Listmonk as the subscriber capture backend for diagnostic completions.

## Operator Notes

Make.com is no longer in the transactional path. Every email trigger is now a direct API call from a Supabase edge function. This gives full code control, Resend log visibility, and removes a paid middleware layer. The diagnostic email is the most important touchpoint — it's the first email a user receives from Worktugal after completing the tool. The copy is minimal and personal by design. Listmonk captures every diagnostic completer into a nurture list with rich attributes (score, segment, country) that can be used for future segmented campaigns.

## Operator Take

Transactional email is now owned infrastructure. Every send is logged in Resend, every subscriber is in Listmonk, and every real-time alert hits Telegram directly. Make.com is out.
