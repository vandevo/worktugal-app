# Thread Handover v2.6 — Ghost Blog Live

## 1. Thread Overview

This session installed Ghost CMS self-hosted on the existing GCP van-cloud server and shipped the first article. The work covered infrastructure (Docker, nginx, SSL), design theming (Emerald Zenith via CSS injection), Ghost Admin API integration into the vault, the full NISS compliance guide written and pushed as a Lexical-formatted post, and a complete image generation pipeline using Gemini API. The session also solved the static 876 user count problem by implementing a live Supabase fetch in the Ghost footer.

The strategic driver was eliminating dependency on third-party publishing platforms while gaining full AI-to-publish automation. Ghost was chosen over Strapi, Sanity, and WordPress because it is free, self-hosted, API-first, and ships with a clean reader experience without needing a custom frontend. Every blog session from now on ends with a draft live in Ghost and a feature image in Downloads — automated end to end.

The session delivered a working content flywheel. The /content command now handles the full BLOG pipeline autonomously: research → write → push to Ghost via API → generate feature image via Gemini → notify Van with the draft URL. The first article (NISS guide) is in Ghost as a draft. Live count is dynamic. app.worktugal.com/blog now redirects to blog.worktugal.com.
