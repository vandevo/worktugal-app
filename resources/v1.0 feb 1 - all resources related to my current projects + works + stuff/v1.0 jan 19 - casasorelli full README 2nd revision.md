## 1. Project Overview

### Purpose

Casa Sorelli is a luxury jewelry marketplace web application designed to showcase and facilitate inquiries for high-end European jewelry pieces. The platform serves as a digital storefront connecting European jewelry designers and houses with discerning collectors.

### Mission

To provide an elegant, premium digital experience for discovering and inquiring about exceptional European jewelry, with seamless communication through WhatsApp and email inquiry forms.

### Target Audience

- Luxury jewelry collectors
- High-net-worth individuals seeking European designer pieces
- Clients interested in Cartier, Bulgari, Buccellati, Chopard, and similar houses

### Operator Contact

- **WhatsApp:** +351 968 138 407
- **Email:** contact@casasorelli.com
- **Phone:** +351 968 138 407

### Key Features

| Feature | Description |
|---------|-------------|
| **Jewelry Catalog** | Filterable catalog with search, category, style, and designer filters |
| **Designer Profiles** | Individual designer pages with logos, descriptions, and country of origin |
| **Product Detail Pages** | Full product information with image galleries, materials, and specifications |
| **Favorites System** | LocalStorage-based favorites allowing users to save pieces without authentication |
| **Inquiry System** | Contact forms that submit to database for admin review |
| **WhatsApp Integration** | Direct WhatsApp links with pre-populated messages for each product |
| **Admin Dashboard** | Protected admin panel for CRUD operations on designers, jewelry, and inquiries |
| **Image Upload** | Direct upload to Supabase Storage for product images and designer logos |
| **Multi-language Support** | Weglot integration for automatic translation |
| **Responsive Design** | Mobile-first design optimized for all viewport sizes |

## 6. Database Schema

### Overview

The database consists of four main tables: `designers`, `jewelry_items`, `inquiries`, and `consignment_submissions`.

Note: See migration `20251217130350_create_consignment_submissions.sql` for the consignment submissions table schema used by the Sell With Us feature.

### Table: designers

Stores information about jewelry designers and luxury houses.

| Column | Type | Nullable | Default | Description |
|--------|------|----------|---------|-------------|
| id | uuid | NO | gen_random_uuid() | Primary key |
| name | text | NO | - | Designer/brand name |
| slug | text | NO | - | URL-friendly identifier (unique) |
| country | text | NO | - | Country of origin |
| logo_url | text | NO | - | Logo image URL |
| description | text | NO | '' | Brand description |
| website_url | text | YES | NULL | Official website |
| featured | boolean | NO | false | Show on homepage |
| display_order | integer | NO | 0 | Sort order for featured |
| created_at | timestamptz | NO | now() | Creation timestamp |
| updated_at | timestamptz | NO | now() | Last update timestamp |

**Indexes:**
- `idx_designers_slug` on slug
- `idx_designers_featured` on featured WHERE featured = true

### Table: jewelry_items

Stores individual jewelry pieces.

| Column | Type | Nullable | Default | Description |
|--------|------|----------|---------|-------------|
| id | uuid | NO | gen_random_uuid() | Primary key |
| designer_id | uuid | NO | - | Foreign key to designers |
| name | text | NO | - | Product name |
| slug | text | NO | - | URL-friendly identifier (unique) |
| category | text | NO | - | Category (Ring, Necklace, etc.) |
| style | text | NO | - | Design style (Modern, Classic, etc.) |
| description | text | NO | '' | Detailed description |
| materials | text | NO | '' | Materials used |
| gemstones | text | NO | '' | Gemstones featured |
| colors_available | text[] | NO | '{}' | Available color options |
| primary_image_url | text | NO | - | Main product image |
| gallery_images | text[] | NO | '{}' | Additional images |
| price_range | text | YES | NULL | Price indication |
| featured | boolean | NO | false | Show on homepage |
| display_order | integer | NO | 0 | Sort order |
| created_at | timestamptz | NO | now() | Creation timestamp |
| updated_at | timestamptz | NO | now() | Last update timestamp |

**Indexes:**
- `idx_jewelry_items_slug` on slug
- `idx_jewelry_items_designer` on designer_id
- `idx_jewelry_items_category` on category
- `idx_jewelry_items_style` on style
- `idx_jewelry_items_featured` on featured WHERE featured = true

**Foreign Keys:**
- `designer_id` references `designers(id)` ON DELETE CASCADE

### Table: inquiries

Stores customer inquiries about jewelry items.

| Column | Type | Nullable | Default | Description |
|--------|------|----------|---------|-------------|
| id | uuid | NO | gen_random_uuid() | Primary key |
| jewelry_item_id | uuid | YES | NULL | Related jewelry item |
| name | text | NO | - | Customer name |
| email | text | NO | - | Customer email |
| phone | text | YES | NULL | Customer phone |
| message | text | NO | - | Inquiry message |
| status | text | NO | 'new' | Status: new, contacted, closed |
| created_at | timestamptz | NO | now() | Submission timestamp |

**Indexes:**
- `idx_inquiries_jewelry_item` on jewelry_item_id
- `idx_inquiries_status` on status

**Foreign Keys:**
- `jewelry_item_id` references `jewelry_items(id)` ON DELETE SET NULL

**Constraints:**
- status CHECK (status IN ('new', 'contacted', 'closed'))

### Storage Buckets

| Bucket | Public | Purpose |
|--------|--------|---------|
| item-images | Yes | Product/jewelry images |
| designer-logos | Yes | Designer brand logos |
| consignment-images | Yes | Consignment submission images |

## 8. Data Flow and Architecture

### User Browse Flow

```
Homepage
    |
    v
Featured Designers/Items (Supabase query with featured=true)
    |
    v
Catalog Page
    |
    v
Filter/Search (client-side filtering)
    |
    v
Product Detail Page
    |
    +--> WhatsApp (external link with pre-filled message)
    |
    +--> Inquiry Form --> Supabase INSERT --> Success Alert
    |
    +--> Add to Favorites --> LocalStorage
```

### Admin Flow

```
Login Page
    |
    v
Supabase Auth (signInWithPassword)
    |
    v
Admin Dashboard
    |
    +--> Jewelry Tab
    |       |
    |       +--> List all items
    |       +--> Add/Edit form --> Supabase INSERT/UPDATE
    |       +--> Delete --> Supabase DELETE
    |
    +--> Designers Tab
    |       |
    |       +--> List all designers
    |       +--> Add/Edit form --> Supabase INSERT/UPDATE
    |       +--> Delete --> Supabase DELETE (cascades items)
    |
    +--> Inquiries Tab
            |
            +--> List all inquiries
            +--> Update status --> Supabase UPDATE
```

### File Upload Flow

```
User selects file
    |
    v
File validated (image type)
    |
    v
supabase.storage.upload() to bucket
    |
    v
Get public URL
    |
    v
URL stored in form state
    |
    v
On save: URL stored in database
```

### Error Handling

- **Network errors:** Caught in try/catch, logged to console, alert shown to user
- **Form validation:** HTML5 required attributes, custom validation before submit
- **Auth errors:** Error message displayed below form
- **Upload errors:** Error message shown below upload component

## 10. Security and Privacy

### Row Level Security (RLS)

**designers table:**
- SELECT: Public (anonymous users can read all)
- INSERT/UPDATE/DELETE: Authenticated only

**jewelry_items table:**
- SELECT: Public (anonymous users can read all)
- INSERT/UPDATE/DELETE: Authenticated only

**inquiries table:**
- SELECT: Not allowed for anonymous (admin only via authenticated session)
- INSERT: Public (anyone can submit inquiries)
- UPDATE/DELETE: Authenticated only

### Storage Policies

**item-images bucket:**
- SELECT: Public
- INSERT/UPDATE/DELETE: Authenticated only

**designer-logos bucket:**
- SELECT: Public
- INSERT/UPDATE/DELETE: Authenticated only

### Authentication

- Email/password authentication via Supabase Auth
- Admin-only access to dashboard
- Session managed via Supabase client

### Input Validation

- HTML5 form validation for required fields
- Email format validation
- File type validation for uploads (image/* only)

### File Upload Restrictions

- Accepted formats: PNG, JPG, WEBP
- Designer logos: Up to 5MB
- Product images: Up to 50MB

### GDPR Considerations

- Inquiry data stored in EU-region Supabase instance (if configured)
- No cookies used except Weglot for translation
- LocalStorage used only for favorites (user preference)

## 11. UX/UI Principles

### Mobile-First Design

- All components designed for mobile viewport first
- Breakpoints: sm (640px), md (768px), lg (1024px), xl (1280px)
- Touch-friendly tap targets (minimum 44x44px)
- Larger buttons and inputs on mobile (py-5 vs py-4)

### Accessibility

- Semantic HTML elements (header, main, footer, nav)
- ARIA labels on icon-only buttons
- Focus states on interactive elements
- Sufficient color contrast (slate-900 on white)
- Keyboard navigation support

### Visual System

**Color Palette:**
